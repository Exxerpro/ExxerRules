# .NET Debugging Tools Reference

## Visual Studio Debugger

### Core Features
- **Breakpoints**: Set, conditional, and tracepoints
- **Watch Windows**: Monitor variables and expressions
- **Call Stack**: Navigate through method calls
- **Immediate Window**: Execute code and evaluate expressions
- **Output Window**: View debug output and diagnostics

### Advanced Features
- **IntelliTrace**: Historical debugging and call stack analysis
- **Diagnostic Tools**: Memory usage, CPU usage, and performance
- **Parallel Stacks**: Debug multithreaded applications
- **Exception Settings**: Configure exception handling behavior

### Memory Debugging
- **Memory Usage**: Real-time memory allocation tracking
- **Heap Analysis**: Object allocation and retention analysis
- **Memory Snapshots**: Compare memory states over time
- **GC Analysis**: Garbage collection behavior monitoring

## JetBrains Rider

### Debugging Features
- **Advanced Breakpoints**: Logpoints, conditional breakpoints
- **Smart Step Into**: Intelligent method stepping
- **Value Tooltips**: Hover over variables for quick inspection
- **Expression Evaluation**: Evaluate complex expressions inline

### Profiling Integration
- **dotMemory**: Integrated memory profiling
- **dotTrace**: Performance profiling and analysis
- **Unit Test Profiling**: Profile individual unit tests
- **Code Coverage**: Visual code coverage analysis

## Command Line Tools

### dotnet-dump
```bash
# Collect memory dump
dotnet-dump collect -p <process-id>

# Analyze memory dump
dotnet-dump analyze <dump-file>

# Common commands
> clrstack -all
> dumpheap -stat
> gcroot <object-address>
> dumpobj <object-address>
```

### dotnet-gcdump
```bash
# Collect GC dump
dotnet-gcdump collect -p <process-id>

# Analyze GC dump
dotnet-gcdump analyze <gcdump-file>
```

### dotnet-trace
```bash
# Start tracing
dotnet-trace collect -p <process-id> --providers Microsoft-DotNETCore-SampleProfiler

# Convert trace to text
dotnet-trace convert <trace-file>
```

### dotnet-counters
```bash
# Monitor performance counters
dotnet-counters monitor -p <process-id>

# List available counters
dotnet-counters list
```

## Logging and Tracing Tools

### Serilog
```csharp
// Configuration
Log.Logger = new LoggerConfiguration()
    .WriteTo.Console()
    .WriteTo.File("logs/app-.txt", rollingInterval: RollingInterval.Day)
    .WriteTo.ApplicationInsights(telemetryConfiguration, TelemetryConverter.Traces)
    .CreateLogger();

// Usage
_logger.LogInformation("Processing request {RequestId}", requestId);
_logger.LogError(ex, "Error processing request {RequestId}", requestId);
```

### System.Diagnostics.Activity
```csharp
// Activity source
private static readonly ActivitySource ActivitySource = new("MyApp");

// Create activity
using var activity = ActivitySource.StartActivity("ProcessRequest");
activity?.SetTag("request.id", requestId);
activity?.SetTag("user.id", userId);
```

## Debugging Best Practices

### Tool Selection
- Choose the right tool for the specific problem
- Use multiple tools for comprehensive analysis
- Combine static and dynamic analysis
- Leverage both local and cloud-based tools

### Data Collection
- Collect data systematically
- Use multiple data sources
- Document the collection process
- Validate data accuracy

### Analysis Process
- Start with high-level analysis
- Drill down to specific issues
- Correlate data from multiple sources
- Document findings thoroughly

### Continuous Improvement
- Learn from each debugging session
- Update tool configurations
- Share knowledge with the team
- Contribute to debugging best practices
