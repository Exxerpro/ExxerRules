# Root Cause Analysis Task

## Objective
Perform comprehensive root cause analysis to identify the underlying cause of a reported issue, not just the symptoms.

## Prerequisites
- Clear problem description with symptoms
- Access to relevant code, logs, and system information
- Understanding of the system architecture and data flow

## Steps

### 1. Problem Definition & Context Gathering
- [ ] Document the exact symptoms and error messages
- [ ] Gather reproduction steps and environment details
- [ ] Identify affected components and their relationships
- [ ] Collect relevant logs, traces, and diagnostic data
- [ ] Document system state at the time of the issue

### 2. Initial Hypothesis Formation
- [ ] Formulate initial hypotheses based on symptoms
- [ ] Identify potential root causes from different angles:
  - Code logic issues
  - Data integrity problems
  - Resource constraints
  - Timing and concurrency issues
  - Configuration problems
  - External dependencies

### 3. Evidence Collection & Analysis
- [ ] Collect and analyze stack traces
- [ ] Review application logs and error patterns
- [ ] Examine performance metrics and resource usage
- [ ] Analyze data flow and state changes
- [ ] Check configuration and environment factors

### 4. Systematic Investigation
- [ ] Trace execution flow through affected code paths
- [ ] Identify the earliest point where the problem manifests
- [ ] Analyze data transformations and state changes
- [ ] Consider timing, concurrency, and resource constraints
- [ ] Test hypotheses with targeted experiments

### 5. Root Cause Identification
- [ ] Determine the primary root cause
- [ ] Identify contributing factors
- [ ] Document the causal chain
- [ ] Validate findings through additional testing

### 6. Solution Design
- [ ] Design fixes that address root causes, not symptoms
- [ ] Consider architectural improvements
- [ ] Plan prevention measures
- [ ] Design monitoring and alerting

### 7. Documentation & Knowledge Transfer
- [ ] Document findings and root cause analysis
- [ ] Explain the solution and its rationale
- [ ] Provide recommendations for prevention
- [ ] Create educational content for the team

## Output
- Comprehensive root cause analysis report
- Detailed solution recommendations
- Prevention strategies and monitoring recommendations
- Educational content for team learning

## Success Criteria
- Root cause is clearly identified and validated
- Solution addresses the underlying issue, not just symptoms
- Prevention measures are implemented
- Team knowledge is enhanced through documentation
